Detailed Project Report: BestCure ERP

Client: Best Cure Medical and General Enterprises  |  Date: February 15, 2026

1. Executive Summary

BestCure ERP is a specialized, web-based Enterprise Resource Planning (ERP) system tailored for veterinary medicine distribution. It acts as the central nervous system of the business, automating key operations:

  • Inventory Management: Real-time tracking of medicine stock, batches, and pricing.
  • Smart Billing (POS): A rapid invoicing interface for walk-in customers and phone orders.
  • Order Management: A portal for clinics to browse products and place orders online.
  • Compliance: Automated generation of professional, GST-compliant invoices with Financial Year numbering (e.g., BC/25-26/0045).

Key Value: The system replaces manual bookkeeping with a secure, cloud-based solution that is accessible from any device, ensuring accuracy and speed.

2. Technical Architecture ("How It Works")

The application is built on the MERN Stack, a modern and robust technology suite. Here's a simplified breakdown:

A. Users (The Frontend)

React.js Tailwind CSS

This is the visible part of the application. It runs in the user's browser (Chrome, Safari). It provides a responsive interface for:

  • Admins: To manage stock, approve orders, and generate bills.
  • Customers: To view products and place orders.

B. The Server (The Backend)

Node.js Express.js

This is the "brain" of the operation. It runs securely on the cloud (Vercel). It handles business logic, such as:

  • Verifying passwords during login (Authentication).
  • Checking if stock is sufficient before confirming an order.
  • Calculating taxes (18% GST).

C. Data Storage (The Database)

MongoDB

This is the digital vault where all data is stored permanently. Unlike spreadsheets, it handles complex relationships (e.g., linking an Order to a specific User and multiple Products) effortlessly.

3. Operational Workflow

Scenario: Generating an Invoice (Admin)

  1. Login: Admin logs in securely via the web portal.
  2. Product Selection: On the billing page, Admin searches for a medicine (e.g., "Amoxicillin"). The system instantly filters 1000+ products.
  3. Cart Building: Admin adds items. The system auto-calculates Subtotal and Tax.
  4. Processing: Admin enters "Customer Name" and clicks "Generate & Print".
  5. Automation:
    • The system checks stock levels.
    • It creates a permanent record in the database.
    • It generates a sequential Invoice Number (e.g., BC/25-26/0023).
    • It marks the order as "Accepted".
  6. Output: A professional PDF invoice is generated instantly for printing.

4. Key Features & Innovations

  • Financial Year Numbering: Invoice numbers reset automatically every fiscal year (April 1st), ensuring strict accounting compliance.
  • Mobile Responsiveness: The entire ERP works seamlessly on smartphones. The sidebar collapses, and complex tables adapt to small screens.
  • Unified Design System: Whether an Admin prints an invoice or a Customer downloads it from their history, the document layout is identical and professional.
  • Role-Based Access Control (RBAC): security is baked in. Customers cannot see Admin dashboards; Admins have full control.

5. Deployment & Maintenance

The application is deployed on Vercel, a serverless platform that ensures 99.9% uptime. Database backups are handled automatically by MongoDB Atlas.
